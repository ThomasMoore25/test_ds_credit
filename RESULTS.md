# RESULTS.md — кратко о подходах и слабых местах

## extract()
Подход: regex-парсеры для суммы (3 числовых формата + сумма прописью),
дат (3 формата → ISO), ИНН (10/12 цифр с отсевом OCR-мусора), контрагента
(ООО/АО/ПАО/ИП). Числовая сумма требует контекста («руб/₽/RUB/сумма/итого/стоимость»),
чтобы не путать её с ИНН/БИК/р/с. Сумма прописью засчитывается без контекста.
Финал — max по всем кандидатам (итоговая сумма обычно наибольшая).
Результат: 5/6 OK + 1 PARTIAL (invoice_002 — спорная дата, см. ниже).

## classify()
Подход: взвешенный keyword scoring, softmax → [0,1], порог top1-top2 = 0.15.
Порог подобран эмпирически: при 0.10 OCR-мусор ошибочно классифицируется;
при 0.20 invoice_002 с коротким текстом сваливается в unknown. 0.15 — оптимум.
Результат: 7/7 OK на датасете.

## check_subject()
Подход: fallback на 9 сельхоз-категорий × keyword + rapidfuzz (fuzzy ≥ 80).
LLM через LangChain — опционально (требует OPENAI_API_KEY), без ключа
работает fallback. Объяснение формируется из совпавшей категории или
запрещённого признака.
Результат: 8/8 PASS + 6/6 FAIL + 3 EDGE (любой исход допустим).

## Слабые места
1. **invoice_002.txt, дата.** README ожидает 2025-02-28 (срок оплаты),
   наша функция возвращает 2025-02-15 (дата документа в шапке). Причина:
   спецификация противоречива — для invoice_001 ожидается именно дата
   документа, а не срок оплаты. Лечится явным маркером «срок оплаты:»
   с приоритетом над «от <дата>», но это сломает invoice_001. Оставлено
   как компромисс.
2. **Сумма прописью.** Словарь ограничен числительными до миллионов.
   «Полтора миллиона», «миллиард» и дробные прописью не поддерживаются.
3. **OCR-мусор.** ИНН с буквами (`770l234567`) намеренно отбрасывается.
   Дата с буквами (`O1.O3.2O25`) восстанавливается не всегда — зависит
   от того, какие символы перепутаны.
4. **EDGE-кейсы в check_subject.** «Юридическое сопровождение сделки» и
   «клининг, уборка административного здания» дают confidence=0.55
   (смешанный случай) — слово «сделки»/«уборка» fuzzy-матчится с категориями.
   Лечится расширением списка запрещённых слов или переключением на LLM.
5. **Классификация — keyword weights.** На новом типе документа (например,
   «доверенность») все скоры будут низкими → unknown. Не плохо, но требует
   доработки при расширении номенклатуры типов.
